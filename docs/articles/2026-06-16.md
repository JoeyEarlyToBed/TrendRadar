
### 20:32-20:38 — SSL 代理问题修复

**问题现象**：
- AI 分析失败：`SSL: UNEXPECTED_EOF_WHILE_READING`（DeepSeek API）
- 企业微信推送失败：同样的 SSL EOF 错误（第 2/3 批次）

**根因**：
- requests 2.33.0 + urllib3 2.5.0 与 Clash HTTP 代理（127.0.0.1:7897）的 HTTPS over HTTP proxy 处理不兼容
- urllib3 2.5.0 直接调用成功，但 requests 2.33.0 封装后失败
- 禁用代理后 requests 也能成功，确认是代理交互问题

**解决方案**：
- 升级 requests 2.33.0 → 2.34.2
- 升级 urllib3 2.5.0 → 2.7.0
- 命令：`python3 -m pip install --upgrade requests urllib3`

**验证结果**：
- litellm 调用 DeepSeek API：✅ 成功
- 企业微信推送 4 批次：✅ 全部成功
- AI 分析完成：✅ 正常
- HTML 报告生成：✅ 正常

**注意**：trendradar 6.9.1 的 requirements.txt 要求 requests==2.33.0，但实际运行 requests 2.34.2 无兼容问题
